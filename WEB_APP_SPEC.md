# Torus Studio: Orbit — Web ミントアプリ仕様書

## このドキュメントについて

このプロジェクトのオーナーとの設計会話をもとに作成した、Webミントアプリの仕様書。
Claude Codeでの実装時に参照すること。

---

## 前提：既存プロジェクトの構造

- Sui Move のNFTプロジェクト
- コントラクトはデプロイ済み（PACKAGE_ID: `0x8971df9b4ea946c47f01baf46ed492ac02290faf8d768b5d4adecd824ed8cbbf`）
- モジュール名: `studio`
- mint関数: `studio::mint(title, desc, image_url, recipient)`
- 現在はシェルスクリプト (`mint_nft.sh`) + `sui client call` でターミナルからミントしている
- 画像はPinataでIPFSにアップロードし、`ipfs://...` リンクをmint時に渡す方式

### 最初に確認すること

`./torus-studio-orbit/sources/studio.move` を読んで以下を確認：
- mint関数の正確なシグネチャ（引数の型と順序）
- Displayテンプレートに設定されている共通フィールド（creator, project_url等）
- アクセス制限の有無（AdminCapなど）

## 作るもの

iPad・スマホ・PCから使える **1ページ完結のミントWebアプリ**。

### 技術スタック

- React + TypeScript
- `@mysten/dapp-kit`（ウォレット接続 + トランザクション構築）
- Vite（ビルド）
- デプロイ先: Vercel等の静的ホスティング（サーバー不要）

### プロジェクト構造

```
~/Desktop/orbit-online/
├── WEB_APP_SPEC.md            ← この指示書
├── torus-studio-orbit/        ← 参考用にクローン（コントラクトの中身確認用）
│   └── sources/studio.move
├── package.json
├── src/                       ← Reactアプリ本体
│   └── ...
└── ...
```

Reactアプリはプロジェクトルート直下に構築する。
`torus-studio-orbit/` はコントラクトの確認用で、Webアプリからは参照しない。

## ページ構成

### ヘッダー
- 左: ロゴ（「Torus Studio」テキスト + 「Orbit mint」サブテキスト）
- 右: ウォレット接続ボタン（接続後はアドレス短縮表示、例: `0x7a3f...c8b2`）

### ミントフォーム（入力フィールド3つ）
1. **タイトル** — テキスト入力。プレースホルダー: 「NFTの名前」
2. **説明** — テキストエリア。プレースホルダー: 「このNFTについての説明...」
3. **画像URL** — テキスト入力。プレースホルダー: 「ipfs://... または https://...」
   - 下にヒントテキスト: 「事前にPinataなどでIPFSにアップロードしてください」

### ミントボタン
- ラベル: 「ミントする」
- ウォレット未接続時は非活性
- 押下時の動作:
  1. `studio::mint` を呼ぶ
  2. `recipient` には接続中のウォレットアドレスを自動で渡す（フォームには表示しない）
  3. ウォレットの署名画面が出る
  4. 成功/失敗をフィードバック表示

### 注記テキスト（ボタン下）
「自分のウォレットにミントされます。送りたい場合はウォレットから転送してください。」

### フッター
接続先ネットワーク + パッケージIDの短縮表示（例: `Sui testnet / Package 0x8971...cbbf`）

## 重要な設計判断（決定済み）

1. **ミントのみ、送信はしない** — recipientにはウォレット接続中の自分のアドレスを自動で入れる。相手への転送はウォレットアプリの標準機能で行う。この方がシンプルで、ミント後に「送るかどうか」を後から決められる。

2. **画像はIPFS（オフチェーン）** — フルオンチェーンだとストレージコストが高すぎる。IPFSのCIDベースで改ざん耐性は確保されている。Pinataが落ちても別のピンニングサービスに切り替え可能。

3. **誰でもミント可能** — コントラクトにアクセス制限はない（要確認）。ミントする人が自分のガス代を払うので、オーナーにコストは発生しない。

4. **プリセット機能やプレビュー機能は不要** — シンプルにフォーム1本。

5. **ネットワーク** — まずtestnetで動作確認、後でmainnetに切り替え。
